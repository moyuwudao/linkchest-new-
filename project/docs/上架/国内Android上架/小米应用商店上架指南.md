## 二、构建APK

### 2.1 构建命令

在 WSL 环境中执行：

```bash
wsl -d linkchest-cn -u mayn -- bash /mnt/d/trae_projects/linkchest/project/apps/mobile/build-gradle.sh china
```

> **注意：**
> - 必须使用 `linkchest-cn` WSL实例
> - 禁止使用 `--clean` 参数
> - 构建产物路径：`apps/mobile/android/app/build/outputs/apk/china/release/`

### 2.2 签名配置

APK 必须使用 Release 签名：

| 项目 | 值 |
|------|-----|
| Keystore文件 | `debug.keystore`（项目内 `android/app/debug.keystore`，密码 `android`） |
| 签名算法 | V1 + V2 |
| 应用签名SHA1 | ⚠️ 待填写（用于微信开放平台绑定） |

> **说明**：项目当前 release 构建使用 `debug.keystore`（已检入 Git 仓库），与微信开放平台登记的签名 MD5 `20F46148B72D8E5E5CA23D37A4F41490` 匹配。上线前**建议**改用独立的 release keystore（详见 [BUILD.md](../../rules/BUILD.md)）。

验证签名：

```bash
apksigner verify --verbose apps/mobile/android/app/build/outputs/apk/china/release/*.apk
```

### 2.3 构建产物验证

```bash
# 检查包名
aapt dump badging *.apk | grep package
# 期望: name='com.linkchest.app'

# 检查应用名称
aapt dump badging *.apk | grep application-label
# 期望: application-label: '链藏'

# 检查不含海外域名
grep "linkchest.net" index.android.bundle
# 期望: 无匹配
```

## 三、上传发布

### 3.1 登录小米开放平台

1. 访问 https://dev.mi.com
2. 使用已注册的开发者账号登录
3. 进入「应用分发」→「应用上架」

### 3.2 填写应用信息

| 字段 | 填写内容 | 参考文档 |
|------|---------|---------|
| 应用名称 | 链藏 | [产品基础信息.md](../产品基础信息.md) |
| 包名 | com.linkchest.app | - |
| 版本号 | 构建时的 versionCode | - |
| 分类 | 效率工具 | - |
| 短描述 | 你的数字收藏宝库 | ≤10字 |
| 长描述 | 见产品基础信息详细介绍 | ≤1000字 |
| 关键词 | 收藏,链接管理,数字资产,笔记,书签,整理,分享,备份,同步 | - |
| 隐私政策URL | https://linkchest.cn/privacy | - |
| 客服邮箱 | support@linkchest.cn | - |

### 3.3 上传材料

| 材料 | 规格 | 数量 |
|------|------|------|
| 应用图标 | 512x512 PNG | 1张 |
| 应用截图 | 1080x1920 或 1080x2400 | 3-5张 |
| APK文件 | 已签名Release APK | 1个 |

### 3.4 上传软著证书

- 格式：PDF扫描件
- 路径：后台「资质管理」→「软件著作权」

### 3.5 测试账号（如需）

如果应用需要登录才能使用核心功能：
- 准备一个测试账号（非管理员账号）
- 在提交审核时备注测试账号信息

## 四、审核要求

### 4.1 必需资质

| 资质 | 状态 | 说明 |
|------|------|------|
| 软件著作权 | ✅ 已有 | 必须提供 |
| ICP备案 | ⬜ 待填写 | 建议提供备案编号 |
| 隐私政策 | ✅ 已上线 | 必须可访问 |

### 4.2 内容分级

- 选择「全年龄」或「4+」
- 确保应用内无暴力、色情、赌博等内容

### 4.3 权限声明

在应用描述或单独字段中声明所需权限：

| 权限 | 用途说明 |
|------|---------|
| 网络访问 | 同步收藏数据 |
| 相机 | 拍照上传图片 |
| 存储 | 读取相册上传图片 |

## 五、注意事项

### 5.1 常见被拒原因

| 原因 | 解决方案 |
|------|---------|
| APK未签名或签名错误 | 确保使用Release签名 |
| 包名与应用名称不匹配 | 包名固定，应用名称填「链藏」 |
| 隐私政策无法访问 | 确认URL有效且可公开访问 |
| 应用崩溃 | 提交前充分测试 |
| 权限申请未说明 | 在描述中说明权限用途 |
| 含有第三方SDK未声明 | 检查 build.gradle 依赖 |

### 5.2 审核周期

| 阶段 | 预计时间 |
|------|---------|
| 首次提交审核 | 3-7个工作日 |
| 版本更新审核 | 1-3个工作日 |
| 被拒后重新提交 | 1-3个工作日 |

### 5.3 版本更新

1. 构建新版本APK（增加 versionCode）
2. 登录小米开放平台
3. 进入应用管理 → 上传新版本
4. 填写更新说明
5. 提交审核
