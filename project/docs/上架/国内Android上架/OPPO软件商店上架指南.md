## 二、构建APK

### 2.1 构建命令

与小米应用商店共用同一APK：

```bash
wsl -d linkchest-cn -u mayn -- bash /mnt/d/trae_projects/linkchest/project/apps/mobile/build-gradle.sh china
```

### 2.2 签名配置

| 项目 | 值 |
|------|-----|
| Keystore文件 | `release-keystore.jks`（项目内管理） |
| 签名算法 | V1 + V2 |
| OPPO应用签名SHA1 | ⚠️ 待填写（OPPO后台需绑定） |

### 2.3 构建产物验证

同小米应用商店，验证包名为 `cn.linkchest.app`。

## 三、上传发布

### 3.1 登录OPPO开放平台

1. 访问 https://open.oppomobile.com
2. 使用已注册的开发者账号登录
3. 进入「应用分发」→「创建应用」

### 3.2 填写应用信息

| 字段 | 填写内容 | 备注 |
|------|---------|------|
| 应用名称 | 链藏 | ≤15字 |
| 包名 | cn.linkchest.app | - |
| 版本号 | 构建时的 versionCode | - |
| 分类 | 效率工具 | - |
| 一句话介绍 | 你的数字收藏宝库 | ≤20字 |
| 应用描述 | 见产品基础信息详细介绍 | ≤3000字 |
| 关键词 | 收藏 链接管理 数字资产 笔记 书签 整理 分享 备份 同步 | 空格分隔 |
| 隐私政策URL | https://linkchest.cn/privacy | 必须可访问 |
| 客服联系方式 | support@linkchest.cn | - |

### 3.3 上传材料

| 材料 | 规格 | 数量 |
|------|------|------|
| 应用图标 | 512x512 PNG | 1张 |
| 应用截图 | 1080x1920 PNG/JPEG | 3-5张 |
| APK文件 | 已签名Release APK | 1个 |

### 3.4 上传软著证书

- 格式：PDF扫描件
- 路径：后台「资质管理」→「软件著作权」

### 3.5 测试账号

如果应用核心功能需要登录：
- 准备测试账号并在审核备注中提供

## 五、注意事项

### 5.1 OPPO特殊要求

| 要求 | 说明 |
|------|------|
| APK大小限制 | 单包≤100MB，超过需分包 |
| 应用名称 | 不可包含「免费」「第一」等夸大词汇 |
| 图标规范 | 不得含有角标、推荐标识等 |

### 5.2 常见被拒原因

| 原因 | 解决方案 |
|------|---------|
| APK签名与备案不一致 | 重新签名并验证 |
| 应用描述含敏感词 | 移除夸大/绝对化用语 |
| 截图含其他品牌标识 | 清理截图中的第三方品牌元素 |
| 隐私政策内容不完整 | 确保包含数据收集说明 |

### 5.3 审核周期

| 阶段 | 预计时间 |
|------|---------|
| 首次提交审核 | 3-5个工作日 |
| 版本更新审核 | 1-3个工作日 |
