## 状态说明

| 状态 | 含义 |
|------|------|
| **待讨论** | 已提出方案，尚未正式评审 |
| **已评审** | 已完成评审，有明确结论（推进/暂缓/放弃） |
| **已排期** | 已决定实施，排入开发计划 |
| **已放弃** | 评审后决定不实施，记录原因备查 |

## 新提案规范

新增优化方案时，按以下规范操作：

1. 在本目录下创建 `{方案名称}.md` 文件
2. 文档内容包含：
   - 需求背景（痛点描述）
   - 技术可行性要点
   - 合规性风险（如有）
   - 待决策事项
3. 在本 README 的方案列表中新增一行
4. 不立即开发，先收集沉淀

---

## 评审流程

1. 定期评审（建议每月一次）
2. 评审维度：用户价值、技术可行性、合规风险、实施成本
3. 评审结论更新状态字段
4. 决定推进的方案转入开发排期
