# 封面缓存至 COS 方案

> 提出时间：2025-05-15 | 状态：待讨论 | 优先级：中

---

## 1. 需求背景

### 1.1 痛点描述

用户在不同设备/网络环境下访问收藏封面时，存在以下问题：

| 场景 | 问题 | 影响 |
|------|------|------|
| 电脑端开 VPN | 可正常访问境外平台封面（YouTube、Facebook 等） | 正常 |
| 手机端未开 VPN | 无法直接访问境外平台封面 URL，封面显示失败 | 体验差 |
| 跨设备同步 | 本地缓存（IndexedDB/FileSystem）不跨设备 | 每台设备需重新缓存 |
| 原始 URL 过期 | TikTok/抖音签名 URL 24h 过期，FB/IG 1-2天过期 | 封面永久丢失 |

### 1.2 核心诉求

将各平台封面统一缓存至国内 COS（腾讯云对象存储），确保任何设备、任何网络环境下都能正常显示封面。

### 1.3 功能范围

- 旗舰版用户专属：仅 `super` tier 用户启用封面缓存功能
- 全平台覆盖：小红书、YouTube、Facebook、Instagram、Twitter/X、TikTok、B站、抖音等所有支持平台
- 缓存至 COS：将平台原始封面 URL 下载并上传至腾讯云 COS，返回 CDN 加速后的缓存 URL

## 2. 技术可行性评估

### 2.1 各平台服务端下载可行性

| 分类 | 平台 | CDN 域名 | URL 过期 | 签名 | Referer 检查 | 可行性 |
|------|------|----------|----------|------|-------------|--------|
| A类 | YouTube | i.ytimg.com | 不过期 | 无 | 无 | ★★★★★ |
| A类 | Twitter/X | pbs.twimg.com | 不过期 | 无 | 有(2023+) | ★★★★☆ |
| B类 | Bilibili | hdslb.com | 不过期 | 无 | 有 | ★★★☆☆ |
| B类 | 小红书 | xhscdn.com | 不过期 | 无 | 有 | ★★★☆☆ |
| C类 | TikTok | tiktokcdn.com | ~24h | 有 | 有 | ★★☆☆☆ |
| C类 | 抖音 | douyinpic.com | ~24h | 有 | 有 | ★★☆☆☆ |
| D类 | Facebook | fbcdn.net | 1-2天 | 有 | 有 | ★☆☆☆☆ |
| D类 | Instagram | fbcdn.net | 1-2天 | 有 | 有 | ★☆☆☆☆ |

### 2.2 关键技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 签名 URL 过期（TikTok/抖音/FB/IG） | 获取 URL 后若不立即下载，URL 失效 | 过期型平台：获取 URL 后同步下载 |
| 防盗链 Referer 检查（B站/小红书/Twitter） | 服务端直接下载返回 403 | 设置对应平台 Referer + UA Header |
| 反爬升级 | 平台随时可能加强反爬 | 监控下载成功率，及时更新 Header 策略 |
| 并发下载被封 IP | 短时间大量请求触发 IP 封锁 | 按平台独立限速队列，指数退避重试 |

### 2.3 推荐技术方案：混合缓存策略

- **A/B 类平台**（YouTube、Twitter、B站、小红书）：异步队列下载 + COS 转存，覆盖 80%+ 用户场景
- **C 类平台**（TikTok、抖音）：获取 URL 后即时同步下载，失败降级到原始 URL
- **D 类平台**（Facebook、Instagram）：不缓存，直接返回原始 URL

### 2.4 数据库设计

新增 `CoverCache` 表，存储原始 URL 与 COS 缓存 URL 的映射：

```prisma
model CoverCache {
  id              String   @id @default(uuid())
  originalUrl     String
  originalUrlHash String   @unique
  cosKey          String
  cosUrl          String?
  urlExpiresAt    DateTime?
  platform        String
  size            Int
  status          String   @default("pending")
  failReason      String?
  retryCount      Int      @default(0)
  expiresAt       DateTime?
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  @@index([originalUrlHash])
  @@index([status])
  @@index([expiresAt])
}
```

## 3. 合规性风险

### 3.1 平台 ToS 风险

| 平台 | 仅引用 URL | 转存至 COS | 关键条款 |
|------|-----------|-----------|---------|
| YouTube | 低风险 | 🔴 高 | API ToS 明确禁止缓存音视频元数据 |
| Facebook | 低风险 | 🔴 高 | Platform Policy 明确禁止 store |
| Instagram | 中风险 | 🔴 高 | 禁止 crawl/scrape + 禁止 store |
| Twitter/X | 低风险 | 🟠 中高 | Developer Agreement 限制缓存时长 |
| TikTok | 低风险 | 🔴 高 | 明确禁止 download/copy/store |
| 小红书 | 中风险 | 🔴 高 | 明确禁止抓取和复制 |
| Bilibili | 低风险 | 🟠 中高 | 有 oEmbed 但禁止复制 |
| 抖音 | 中风险 | 🔴 高 | 明确禁止复制和下载 |

**核心结论**：所有 8 个平台的 ToS 均不授权第三方将封面图转存至自有存储。

### 3.2 中国法规风险

| 法规 | 风险等级 | 说明 |
|------|---------|------|
| 《网络安全法》第47条 | 🔴 最高 | 转存后从"链接索引服务"变为"内容存储服务"，需承担内容审核义务 |
| 《信息网络传播权保护条例》 | 🟠 高 | 转存后"避风港"保护可能失效 |
| 《网络信息内容生态治理规定》 | 🟠 高 | 需对存储内容建立审核机制 |

### 3.3 必要合规措施

| 措施 | 优先级 | 说明 |
|------|--------|------|
| 接入图片审核 API（腾讯天御/阿里绿网） | P0 | 每张缓存图片上传前自动审核 |
| 用户协议更新 | P0 | 增加封面缓存条款 |
| 版权声明与投诉机制 | P1 | 提供版权投诉入口 |
| 仅缓存用户已收藏的封面 | P1 | 不主动批量爬取 |
| 保留原始 URL 映射 | P1 | 可追溯图片来源 |
| 设置缓存有效期（30天） | P2 | 减少长期存储的合规风险 |

## 4. 待决策事项

### 4.1 是否推进

| 选项 | 说明 |
|------|------|
| **推进** | 接入图片审核 API + 更新用户协议 + 分阶段实施混合缓存 |
| **暂缓**（当前决策） | 维持现有本地缓存方案，纳入优化方案池后续讨论 |
| **放弃** | 不做 COS 缓存，仅优化本地缓存体验 |

### 4.2 推进条件

若后续决定推进，需满足以下前置条件：

1. **合规准备完成**：图片审核 API 接入、用户协议更新、版权投诉机制建立
2. **用户反馈验证**：确认跨设备/跨网络封面问题是高频痛点（而非低频边缘场景）
3. **成本预算确认**：图片审核 API 费用（首月 ~¥105，稳态 ~¥10/月）
4. **D 类平台策略确定**：Facebook/Instagram 是否需要覆盖

### 4.3 替代方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **增强本地缓存**（当前维持） | 零服务端成本、无合规风险 | 不解决跨设备问题 |
| **Nginx 反向代理** | 无需存储 | 仍受防盗链限制、带宽成本高 |
| **COS 图片处理+样式参数** | 不下载原文件 | 仍需下载原文件到 COS |
| **PWA Service Worker 优化** | 客户端离线缓存 | 不解决跨设备问题 |
