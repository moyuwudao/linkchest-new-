# LINKCHEST 平台优先级分级策略

> 版本: v3.1 | 日期: 2026-04-22 | 平台总数: 91（已从 193 精简）

---

## 一、分级背景与目标

### 1.1 目标用户画像

| 维度 | 描述 |
|------|------|
| **核心动机** | 有收藏欲望、需要后续回顾 |
| **使用场景** | 多平台内容收集（学习/旅游/美食参考） |
| **市场定位** | 国内国外同等重要，优先上架 Google Play |

### 1.2 分级目的

- **聚焦资源**：将研发、运营、推广资源集中在高价值平台
- **优化体验**：在添加收藏页面按优先级展示平台快速选择
- **降低噪音**：移除低价值/重复配置，减少用户认知负担
- **数据驱动**：通过优先级指导后续的平台深度集成排序

## 二、分级标准

| 级别 | 名称 | 定义 | 数量 | 集成深度 |
|------|------|------|------|----------|
| **S** | 全球核心命脉 | 不做就别做产品，用户最高频使用的平台 | 12 | 完整解析 + Deep Link + 封面提取 |
| **A** | 区域高频 | 各市场用户的高频场景平台 | 23 | 完整解析 + Deep Link |
| **B** | 垂直头部 | 特定人群的高价值平台，各市场都有明确价值 | 26 | 基础解析 + Deep Link |
| **C** | 中低频垂直 | 特定人群，保留但不做重点优化 | 30 | 基础识别 |
| **D** | 已移除 | 纯地域性、工具类、重复配置的低价值平台 | ~102 | 不再支持 |

### 2.1 S 级判定标准

- 月活超过 5 亿或同赛道绝对 TOP1
- 用户收藏行为天然高频（视频/社交/内容）
- 国内+国际市场双覆盖或同生态对标

### 2.2 A 级判定标准

- 月活超过 1 亿或垂直赛道 TOP3
- 对应目标用户的高频场景（生活/电商/音乐/旅游）
- 在主要市场（中/美/欧）有明确用户基础

### 2.3 B 级判定标准

- 特定人群的高价值平台（开发者/设计师/学习者/游戏玩家）
- 虽非全民应用，但目标用户渗透率极高
- 有明确的收藏/参考使用场景

### 2.4 C 级判定标准

- 有用户需求但使用频率较低
- 垂直细分领域的中等平台
- 保留以覆盖长尾需求，不做重点优化

### 2.5 D 级移除标准

- 纯地域性小平台（仅在单一国家/地区运营）
- 与已有平台功能高度重复
- 工具类/企业类平台，非内容消费场景
- 配置重复（如 Threads 与 Instagram 重复，Stack Overflow 配置重复）

## 五、已移除平台（D 级，约102个）

### 5.1 移除原因分类

| 原因 | 示例 | 说明 |
|------|------|------|
| **纯地域性** | Zalo(VN)、KakaoTalk(KR)、Line(JP) | 仅在单一国家运营 |
| **功能重复** | Threads、LinkedIn 重复配置 | 与已有平台高度重叠 |
| **工具/企业类** | Salesforce、Jira、Zoom | 非内容消费场景 |
| **低频小众** | 各类独立博客平台、小众论坛 | 用户量不足 |
| **配置重复** | Stack Overflow 多处重复 | 代码维护问题 |

### 5.2 典型移除平台列表

```
// 纯地域性
Zalo, KakaoTalk, Line, VK, Odnoklassniki, Naver, Daum

// 工具/企业类
Salesforce, Jira, Confluence, Zoom, Google Meet, Notion(重复)

// 功能重复
Threads (同 Instagram), Tumblr (同 Twitter/微博)

// 低频小众
Vimeo(视频,次于YT), Dailymotion, Mix, Diigo

// 其他
MyAnimeList, Goodreads, Letterboxd, Strava, Fitbit, Duolingo
```
