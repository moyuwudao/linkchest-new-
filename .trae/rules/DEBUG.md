## 2. 日志规范

### 2.1 日志级别

| 级别 | 用途 | 示例 |
|------|------|------|
| **DEBUG** | 详细调试信息 | 函数调用、参数值 |
| **INFO** | 一般信息 | 服务启动、请求处理 |
| **WARN** | 警告信息 | 潜在问题、不推荐使用 |
| **ERROR** | 错误信息 | 异常、失败操作 |
| **FATAL** | 致命错误 | 系统崩溃、严重问题 |

### 2.2 日志格式

```typescript
// 日志格式示例
logger.info('User logged in', {
  userId: '123',
  timestamp: new Date().toISOString(),
  ip: '192.168.1.1'
});
```

### 2.3 日志输出

| 环境 | 日志级别 | 输出位置 |
|------|----------|----------|
| **开发环境** | DEBUG+ | 控制台 |
| **测试环境** | INFO+ | 控制台 + 文件 |
| **生产环境** | WARN+ | 文件 + 监控 |

## 3. 问题排查流程

### 3.1 通用排查步骤

```
1. 重现问题
    ↓
2. 收集日志
    ↓
3. 定位问题位置
    ↓
4. 分析根本原因
    ↓
5. 提出修复方案
    ↓
6. 验证修复效果
```

### 3.2 常见问题排查

| 问题类型 | 排查步骤 |
|----------|----------|
| **API 错误** | 检查网络请求 → 查看服务器日志 → 检查数据库状态 |
| **前端错误** | 查看浏览器控制台 → 检查网络请求 → 检查组件状态 |
| **构建错误** | 查看构建日志 → 检查依赖版本 → 检查配置文件 |
| **数据库错误** | 检查连接配置 → 查看数据库日志 → 验证 SQL 语句 |

## 6. 调试最佳实践

### 6.1 调试原则

| 原则 | 说明 |
|------|------|
| **最小可复现** | 找出最小的复现步骤 |
| **日志追踪** | 在关键位置添加日志 |
| **逐步缩小** | 从大范围逐步缩小到问题点 |
| **验证假设** | 每次只验证一个假设 |

### 6.2 调试技巧

| 技巧 | 说明 |
|------|------|
| **二分法** | 将代码分成两部分，确定问题在哪部分 |
| **回滚测试** | 逐步回滚代码，找到引入问题的提交 |
| **隔离测试** | 将问题代码独立出来测试 |
| **对比测试** | 对比正常和异常情况的差异 |

## 7. 常见错误及解决方案

### 7.1 网络错误

| 错误 | 解决方案 |
|------|----------|
| `ECONNREFUSED` | 检查服务是否启动、端口是否正确 |
| `ETIMEDOUT` | 检查网络连接、服务器是否可达 |
| `CORS` | 检查 CORS 配置 |

### 7.2 数据库错误

| 错误 | 解决方案 |
|------|----------|
| `Connection refused` | 检查数据库服务是否启动 |
| `Authentication failed` | 检查用户名密码 |
| `Table not found` | 运行数据库迁移 |

### 7.3 代码错误

| 错误 | 解决方案 |
|------|----------|
| `ReferenceError` | 检查变量是否定义 |
| `TypeError` | 检查类型是否正确 |
| `SyntaxError` | 检查语法错误 |
