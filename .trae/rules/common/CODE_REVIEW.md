# CODE_REVIEW.md — 代码审查标准

> 本文档定义代码审查的标准流程、检查清单和问题严重性分级，确保代码质量和安全性。

---

## 1. 审查流程

### 1.1 标准流程

```
┌─────────────────────────────────────────────────────────────┐
│                    代码审查流程                             │
├─────────────────────────────────────────────────────────────┤
│  1. 作者自检 → 2. 自动化检查 → 3. 人工审查 → 4. 反馈修复    │
│       ↓              ↓              ↓            ↓          │
│  lint/测试    CI 自动检查     代码审查      代码修复        │
│                           安全审查                          │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 审查入口条件

| 条件 | 说明 |
|------|------|
| ✅ 代码已提交到 feature 分支 |
| ✅ 所有 lint 检查通过 |
| ✅ 测试覆盖率 ≥ 80% |
| ✅ 构建成功 |

## 3. 问题严重性分级

### 3.1 分级标准

| 级别 | 标识 | 含义 | 操作 |
|------|------|------|------|
| **CRITICAL** | 🔴 | 安全漏洞或数据丢失风险 | **BLOCK** - 合并前必须修复 |
| **HIGH** | 🟠 | Bug 或严重质量问题 | **WARN** - 合并前应修复 |
| **MEDIUM** | 🟡 | 可维护性问题 | **INFO** - 考虑修复 |
| **LOW** | 🟢 | 样式或小建议 | **NOTE** - 可选 |

### 3.2 分级示例

| 级别 | 示例问题 |
|------|----------|
| **CRITICAL** | 硬编码密钥、SQL 注入、XSS 漏洞 |
| **HIGH** | 未处理的错误、空 catch 块、类型错误 |
| **MEDIUM** | 重复代码、过长函数、命名不清晰 |
| **LOW** | 空格格式、注释格式、代码组织 |

## 4. 安全触发器

### 4.1 自动触发安全审查的场景

| 场景 | 触发条件 |
|------|----------|
| **认证模块修改** | 修改 auth.ts 或 JWT 相关代码 |
| **密码处理** | 涉及密码存储、验证的代码 |
| **支付相关** | 修改支付流程或金额计算 |
| **用户数据** | 修改用户敏感信息处理 |
| **文件上传** | 修改文件上传功能 |
| **API 接口** | 新增或修改对外 API |

## 6. 审查反馈流程

### 6.1 反馈原则

| 原则 | 说明 |
|------|------|
| **具体** | 指出具体文件和行号 |
| **清晰** | 明确说明问题和影响 |
| **建设性** | 提供修复建议 |
| **尊重** | 使用友好的语言 |

### 6.2 修复确认

```
⚠️ 修复确认
┌─────────────────────────────────────────────┐
│ 问题级别: HIGH                              │
│ 文件: src/users.ts:120                      │
│ 原问题: 未处理的 Promise rejection           │
│ 修复方式: 添加 try-catch 块                 │
└─────────────────────────────────────────────┘
已修复，请确认。 [Y/N]
```

## 7. 工具辅助

### 7.1 推荐 Agent

| Agent | 用途 | 触发时机 |
|-------|------|----------|
| **code-reviewer** | 通用代码审查 | 任何代码改动 |
| **security-reviewer** | 安全审查 | 认证、支付、用户数据 |
| **typescript-reviewer** | TypeScript 审查 | TS/JS 代码 |
| **build-error-resolver** | 构建错误修复 | 构建失败时 |

### 7.2 自动审查流程

```
1. PR 创建 → 自动运行 lint + typecheck
2. 测试通过 → 自动调用 code-reviewer
3. 涉及安全 → 自动调用 security-reviewer
4. 审查完成 → 生成报告
```
