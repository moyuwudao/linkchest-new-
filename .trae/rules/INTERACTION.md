# INTERACTION.md — 交互基础规则

> 本文档定义 AI 助手与开发者（你）之间的沟通方式、问问题的时机和确认机制。
> **目标行数：≤150 行**

---

## 1. 沟通方式

### 1.1 语言
- **默认语言**：中文
- **代码注释**：中文（与项目现有风格一致）
- **技术术语**：中英对照（如：API接口、TypeScript类型）

### 1.2 响应风格
- **日常任务**：直接给代码 + 简短注释说明
- **设计讨论**：方案对比 + 简要说明（为什么用、好处、影响）
- **学习场景**：结合实践解释概念，提供示例代码

### 1.3 提问方式

| 场景 | 推荐方式 |
|------|----------|
| **需求描述** | 用自然语言描述想要实现的功能或解决的问题 |
| **代码问题** | 直接贴代码或描述具体错误信息 |
| **学习请求** | 说明想了解的技术点或概念 |
| **Bug报告** | 描述现象 + 复现步骤 + 期望行为 |

## 2. 问问题的时机

### 2.1 何时提问（AI → 用户）

| 场景 | 是否提问 | 说明 |
|------|----------|------|
| **需求模糊** | ✅ 是 | 当需求描述不清晰时，会追问确认 |
| **方案选择** | ✅ 是 | 列出 2-3 个方案，给出推荐并请你确认 |
| **重大变更** | ✅ 是 | 修改环境配置、删除代码、升级依赖等需确认 |
| **技术选型** | ✅ 是 | 引入新技术或库时，说明利弊并请你决定 |
| **潜在风险** | ✅ 是 | 在任务范围内发现安全隐患、性能问题时指出（参见 SOUL.md §2.4） |
| **构建APK** | ✅ 是 | 强制触发 [HIGH_RISK.md](HIGH_RISK.md) 第6章（构建安全）阻断机制 |
| **部署服务** | ✅ 是 | 强制触发 [HIGH_RISK.md](HIGH_RISK.md) 第2章（部署安全）阻断机制 |

### 2.2 何时直接执行（AI → 用户）

| 场景 | 是否直接执行 | 说明 |
|------|--------------|------|
| **明确的代码编写** | ✅ 是 | 根据清晰需求直接写代码 |
| **代码审查反馈** | ✅ 是 | 直接指出问题 + 影响 + 修复建议 |
| **日常维护任务** | ✅ 是 | 格式化代码、添加注释、小改进 |
| **文档整理** | ✅ 是 | 整理规则文档、README 等 |

## 3. 操作确认机制

### 3.1 必须确认的操作

| 操作类型 | 示例 | 确认方式 |
|----------|------|----------|
| **删除文件** | 删除整个目录或多个文件 | 列出文件清单，确认后执行 |
| **大量代码变更** | 重构超过 200 行代码 | 说明变更范围和影响 |
| **修改配置** | 修改 .env、build.gradle 等 | 展示变更内容 |
| **数据库变更** | Prisma schema 变更、数据迁移 | 说明迁移风险 |
| **依赖升级** | 升级 major 版本或核心依赖 | 说明兼容性风险 |
| **构建APK** | 任何 Gradle/Expo 构建命令 | 强制通过 [HIGH_RISK.md](HIGH_RISK.md) 检查 |
| **部署服务** | 任何 ssh/scp/pm2/prisma 部署命令 | 强制通过 [HIGH_RISK.md](HIGH_RISK.md) 检查 |
| **创建规则文件** | 在 `.trae/rules/` 下创建新规则 | 通过 rule-validator 验证 |

### 3.2 确认格式

```
⚠️ 操作确认
┌─────────────────────────────────────────┐
│ 操作类型：[类型]                        │
│ 影响范围：[文件/模块]                   │
│ 风险等级：[高/中/低]                    │
│ 预期结果：[描述]                        │
└─────────────────────────────────────────┘
确认执行？ [Y/N]
```

## 4. 任务管理

### 4.1 任务状态
- 使用 `TodoWrite` 工具管理多步骤任务
- 每个任务有明确的状态：pending → in_progress → completed
- 每个时间点恰好有一个 `in_progress` 任务
- 完成后立即标记（不要批量标记）

### 4.2 任务中断与恢复
- 可以随时暂停任务
- 恢复时会总结之前的进度
- 支持断点续做

## 6. Skill 与 MCP 工具清单

> Skill 定义、触发条件、MCP 工具清单 → 统一见 [AGENTS.md](common/AGENTS.md)。不在此重复。

**关键原则**：
- 服务器连接优先使用 `aliyun-servers MCP`，降级为 `ssh` 命令
- 页面验证优先使用 `Chrome DevTools MCP`，降级为 `Playwright MCP`
- 数据库查询优先使用 `PostgreSQL MCP`（只读），降级为 `aliyun-servers` + `psql`
- 库 API 查文档优先使用 `Context7 MCP`，降级为 WebSearch
