# CONTEXT.md — 上下文保持规则

> 本文档定义对话上下文的保持策略，确保在长期对话中信息不丢失。

---

## 1. 上下文管理策略

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **相关性优先** | 只保留与当前任务相关的信息 |
| **定期总结** | 每 5-10 轮对话进行一次上下文总结 |
| **关键信息标记** | 对重要信息进行标记和保存 |
| **上下文清理** | 移除过时或无关的信息 |

### 1.2 上下文类型

| 类型 | 说明 | 保留策略 |
|------|------|----------|
| **任务目标** | 用户的核心需求 | 始终保留 |
| **已完成步骤** | 已执行的操作 | 保留最近 3 步 |
| **关键决策** | 重要的技术决策 | 始终保留 |
| **代码片段** | 讨论过的代码 | 保留相关代码 |
| **错误信息** | 遇到的错误 | 保留最近 2 个 |
| **临时信息** | 中间讨论内容 | 任务完成后清理 |

---

## 2. 上下文总结机制

### 2.1 自动总结

每完成一个子任务或经过 5-10 轮对话后，自动生成上下文总结：

```
📋 上下文总结
┌─────────────────────────────────────────┐
│ 任务目标：[描述]                        │
│ 已完成：[步骤列表]                      │
│ 关键决策：[决策列表]                    │
│ 当前状态：[状态描述]                    │
└─────────────────────────────────────────┘
```

### 2.2 手动总结

用户可以随时要求总结：

```
用户：总结一下当前进度
AI 助手：生成上下文总结
```

## 3. 信息保存策略

### 3.1 永久保存

| 信息类型 | 保存位置 |
|----------|----------|
| 规则文档 | `.trae/rules/` |
| 配置文件 | `.trae/config/` |
| 代码模板 | `.trae/templates/` |

### 3.2 临时保存

| 信息类型 | 保存位置 | 有效期 |
|----------|----------|--------|
| 对话历史 | 内存 | 会话期间 |
| 任务状态 | 内存 | 任务完成后 |
| 临时数据 | 内存 | 步骤完成后 |

---

## 4. 上下文恢复

### 4.1 会话恢复

如果会话中断，可以通过以下方式恢复：

```
用户：继续之前的任务
AI 助手：检索历史上下文，恢复任务状态
```

### 4.2 断点续做

支持断点续做，自动定位到上次中断的位置：

```
用户：继续
AI 助手：[恢复上下文] 我们上次做到...
```

---

## 5. 上下文优化

### 5.1 压缩策略

- **代码压缩**：只保留关键代码片段
- **信息去重**：移除重复信息
- **优先级排序**：按重要性排序信息

### 5.2 Token 管理

| 阈值 | 操作 |
|------|------|
| > 80% | 提示用户进行上下文总结 |
| > 90% | 自动进行上下文压缩 |
| > 95% | 强制清理非关键信息 |

---

## 6. 最佳实践

### 6.1 保持简洁

- 避免在对话中包含大量无关信息
- 及时清理临时信息
- 使用简洁的语言描述

### 6.2 明确边界

- 明确当前任务的范围
- 完成任务后进行总结
- 开始新任务前清理上下文
